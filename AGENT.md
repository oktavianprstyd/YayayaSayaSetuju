# 🤖 OpenCode Agent: UMKM Legalitas & Micro-Capital

> **Agent Role:** Fullstack Developer  
> **Design Partner:** Open Design (AI Design Agent)  
> **Mode:** Autonomous (self-iterating with checkpoints)  
> **Project:** UMKM Legalitas → Micro-Capital Crowdfunding  
> **Methodology:** Waterfall (Phase 1 must complete before Phase 2)

---

## 🎨 Open Design Integration

This agent works with **Open Design** (https://open-design.ai/) — an open-source AI design workspace.

### Workflow

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   OPEN DESIGN   │────▶│  DESIGN-BRIDGE  │────▶│  OPENCODE AGENT │
│   (AI Designer) │     │   (This Agent)  │     │  (AI Developer) │
│                 │     │                 │     │                 │
│ • Generate UI   │     │ • Read tokens   │     │ • Implement     │
│ • Export tokens │     │ • Map components│     │ • Code logic    │
│ • Create layout │     │ • Follow specs  │     │ • Build pages   │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### How It Works

1. **User** prompts Open Design → generates visual design
2. **Open Design** exports design tokens + HTML preview
3. **This agent** reads `DESIGN-BRIDGE.md` for design specs
4. **OpenCode** implements design into Next.js + shadcn/ui code

### Design Files

| File | Purpose |
|------|---------|
| `DESIGN-BRIDGE.md` | Design tokens, component mapping, layout specs |
| `design-tokens.json` | (Auto-generated by Open Design) |
| `design-preview.html` | (Auto-generated by Open Design) |

## 📋 Agent Rules (MUST FOLLOW)

### Rule 1: Read Context First
- ALWAYS read `claude.md` before starting ANY task
- ALWAYS read `ARCHITECTURE.md` before writing code
- ALWAYS read `MASTER-PROMPT.md` to know current phase
- **IF DESIGN-BRIDGE.md exists** → read it for design specs

### Rule 2: One Phase at a Time
- Complete Phase 1 100% before touching Phase 2
- Each phase has a CHECKPOINT file — verify it exists before proceeding
- If checkpoint missing, STOP and ask user

### Rule 3: Design-First Approach (with Open Design)
- When implementing UI components, check `DESIGN-BRIDGE.md` first
- Use design tokens (colors, spacing, typography) from bridge file
- Map components to shadcn/ui equivalents as specified
- Follow layout structure exactly

### Rule 4: Mobile-First Design
- All UI must work on 320px width (small phones)
- Use Tailwind responsive prefixes: `sm:`, `md:`, `lg:`
- Test interactions with touch (not just mouse)

### Rule 5: Indonesian Language Only
- ALL user-facing text must be in Bahasa Indonesia
- Error messages, labels, placeholders, tooltips — everything
- No English in UI (code comments can be English)

### Rule 6: No Backend in Phase 1
- Phase 1 uses LocalStorage ONLY
- No API calls, no database, no auth
- All data persists in browser

### Rule 7: Validate Everything
- Use Zod for ALL form validation
- Show Indonesian error messages
- Never trust user input

## 🎮 Agent Commands

### How to Use This Agent

```bash
# Start the agent (Phase 1) - with Open Design
opencode --file AGENT.md --file claude.md --file ARCHITECTURE.md --file MASTER-PROMPT.md --file DESIGN-BRIDGE.md "Start Phase 1 execution"

# Resume from checkpoint
opencode --file AGENT.md --file CHECKPOINT.md "Continue from last checkpoint"

# Specific task with design specs
opencode --file AGENT.md --file DESIGN-BRIDGE.md "Implement Navbar with design tokens"

# Debug
opencode --file AGENT.md "Fix error: [paste error]"
```

### Agent Response Format

When completing a task, the agent MUST:

1. **Report what was done**
2. **Show file changes**
3. **Reference design tokens used** (if DESIGN-BRIDGE.md exists)
4. **Update CHECKPOINT.md**
5. **Suggest next action**

Example:
```
✅ TASK COMPLETED: Navbar component

Design tokens used (from DESIGN-BRIDGE.md):
- Color: --color-primary: #059669
- Spacing: --space-4: 1rem
- Font: --font-sans: 'Inter'

Files created/modified:
- src/components/layout/Navbar.tsx (new)
- src/app/layout.tsx (modified)

CHECKPOINT updated: Phase 1.2, Task 2/3 complete

Next: Implement Footer component
Suggested command: opencode --file AGENT.md --file DESIGN-BRIDGE.md "Implement Footer"
```

## 🔍 Agent Self-Check List

Before marking any task complete, verify:

- [ ] Code compiles without errors (`npm run build` passes)
- [ ] No TypeScript errors (`npx tsc --noEmit` passes)
- [ ] Component renders correctly in browser
- [ ] Mobile responsive (test 320px, 768px, 1024px)
- [ ] All text in Bahasa Indonesia
- [ ] Zod validation working
- [ ] LocalStorage persistence working
- [ ] Accessibility: proper labels, focus states, ARIA
- [ ] Loading states implemented
- [ ] Error handling with try/catch
- [ ] **Design tokens from DESIGN-BRIDGE.md applied correctly**

## 🚨 Agent Stop Conditions

STOP and ask user if:

1. **CHECKPOINT.md is missing** and user wants to skip phases
2. **Phase 1 not complete** but user asks for Phase 2 features
3. **External API needed** in Phase 1 (against rules)
4. **User wants to change tech stack** mid-project
5. **Security concern found** (exposing sensitive data)
6. **Build fails** after 3 fix attempts
7. **User asks for feature not in requirements**
8. **DESIGN-BRIDGE.md missing** but user expects design-spec implementation
